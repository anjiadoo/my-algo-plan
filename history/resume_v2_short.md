## 技术视野 · 智驾数据闭环
> 主动学习自动驾驶技术演进与数据闭环工程实践，尝试把互联网侧的高并发、实时数据、多存储选型经验对应到智驾工程域；目前尚无智驾方向实际项目经验，以下为学习整理后的理解框架。

+ **技术演进 & 战略框架**：智驾经历"规则驱动 → 模块化 ADAS → CNN → BEV+Transformer → 端到端 → VLA / 世界模型"六次范式跃迁，2023 后行业核心竞争维度从"拼算法"迁移到"拼数据闭环转速"。行业以 **"一个飞轮（数据闭环）+ 两条腿（规模化量产 / 前沿技术制高点）"** 展开竞争——三者互为因果，缺一不可。
+ **数据闭环全景**：完整闭环是跨端-边-云的大规模数据流水线，可拆为八环——触发（影子模式）→ 采集上传 → 入湖 + 元数据 → 挖掘（规则 / 向量 / VLM 三代）→ 4D 自动标注 → 训练（加权采样）→ 仿真验证（回灌 + 世界模型场景生成）→ 灰度 OTA。三个转速指标（T1<24h / T2<7d / T3<14d）构成第一梯队的工程线。
+ **后端工程侧映射**：智驾数据闭环的核心痛点是互联网侧问题的变体——海量存储检索（对象存储 + 数据湖 + 多存储分工）、长尾场景挖掘（规则 / 向量 / VLM 三代 = "用哪种索引匹配哪种查询语义"）、数据版本管理（DVC + Iceberg Snapshot = 数据即代码）、训练供给（WebDataset + Alluxio 破 IOPS 瓶颈 = 按物理约束反推架构）。
